# PR Title & Description Convention

Roland autogenerates draft pull requests when executor tasks complete the task git workflow (`src/rco/task-git-workflow.ts`) or when a **closed-loop harness** finishes (`src/loop-engine/closed-loop.ts`). Titles and bodies follow a single clean convention — no mission noise, no agent boilerplate.

---

## PR titles

**Format:** `type(scope): short imperative description`

| Part | Rule |
|------|------|
| `type` | Inferred from task wording: `feat` (default), `fix`, `refactor`, `docs`, `test`, `chore` |
| `scope` | Inferred area: `dashboard`, `git`, `loop`, `pm`, `mcp`, `api`, `auth`, or `roland` |
| `description` | Task title with mission noise stripped; lowercase imperative; max 72 chars total |

### Before / After

| Before (legacy — auto-migrated) | After (clean) |
|----------------------------------|---------------|
| `Task task-1: [Mission: PR-cleanup] Team Goal: Improve PR Title…` | `feat(git): improve pr title and description convention` |
| `Roland: Add rate limiting to the Express API` | `feat(api): add rate limiting to express api` |
| `Task-3: Fix dashboard mobile layout on iPhone` | `fix(dashboard): restore mobile layout on iphone` |
| `[Mission: Loop Harness] Implement evaluation gate retry logic` | `feat(loop): implement evaluation gate retry logic` |

### Good examples

```
feat(git): implement clean pr title and description convention
fix(dashboard): restore mobile responsiveness on iphone
refactor(loop): simplify critique escalation logic
docs(guide): add closed-loop harness documentation
test(loop): add exit condition evaluation coverage
chore(deps): bump vitest to latest minor
```

### Avoid

- `Task task-N:` prefixes
- `[Mission: …]` brackets
- `Roland:` prefixes
- PM mandate blocks pasted into titles
- Titles longer than 72 characters

## PR descriptions

Autogenerated bodies include structured sections:

1. **Summary** — one paragraph from the task description (noise stripped)
2. **Key changes** — bullet list from task scope
3. **Testing** — Roland workflow notes; Vanguard may append specifics via `testingNotes`
4. **Related** — task id, agent, mission goal, run id, dashboard URL
5. **Impact** *(optional)* — why the change matters when `impactNote` is provided

### Before / After (body)

**Before** — legacy `## Roland Run` template with agent handoff noise:

```markdown
## Roland Run
**Goal:** Task task-1: [Mission: PR-cleanup] Team Goal: Improve PR Title…

⚠️ PRODUCTION HARDENING MANDATE
- [ ] All tests pass
- [ ] No console.log in production paths
…

## Sparrow Handoff Protocol
Generated automatically by Roland
```

**After** — clean structured body:

```markdown
## Summary
Implements conventional-commit PR titles and structured bodies for Roland task git workflow and closed-loop completion.

## Key changes
- Add `formatPrFromGoal()` and `buildPrDescription()` in pr-format.ts
- Strip mission noise from legacy titles via `stripMissionNoise()`
- Wire clean titles into task-git-workflow and closed-loop PR output

## Testing
- Unit tests in tests/unit/pr-format.test.ts
- Vanguard test-executor ran `npm run test:run` — all green

## Related
- Task: task-1 · Agent: executor
- Goal: Improve PR title convention
- Run: run-abc123
```

Implementation: `buildPrDescription()` and `formatPrFromGoal()` in `src/rco/pr-format.ts`.

## Commit messages

Per-task commits use the same conventional title as the PR, plus:

```
feat(api): add rate limiting to password reset endpoint

Closes task task-1

Generated by Roland
```

## Retroactive cleanup

Rename recent open PRs that still use legacy titles or bodies:

```bash
# Preview changes (default — dry-run)
roland pr-cleanup

# Apply title renames via GitHub CLI
roland pr-cleanup --apply

# Current branch PR only
roland pr-cleanup --current --apply

# Also migrate legacy "## Roland Run" bodies
roland pr-cleanup --body --apply

# Scan fewer PRs
roland pr-cleanup --limit 10 --apply
```

Requires `gh` authenticated against the repo. PRs already matching the convention are skipped.

### Standalone script

```bash
npm run build && node scripts/pr-cleanup.ts --apply

# Preview title/body from a goal without gh
node scripts/pr-cleanup.ts --goal "Implement rate limiting for the API"
```

## Dashboard & web integration

The Roland Web server (`roland-web/server/routes/projects.ts`) uses the same formatters for push-and-create-PR flows. The dashboard applies `suggestPrCleanup()` when displaying PR metadata for connected GitHub repos.

---

## Manual migration

When batch rename is not available, edit the title in GitHub to:

```
type(scope): <stripMissionNoise(original title)>
```

Preview a suggested title in Node:

```typescript
import { formatPrFromGoal, migrateLegacyPrTitle } from './dist/rco/pr-format.js';

formatPrFromGoal('Fix dashboard mobile layout on iPhone');
// → { title: 'fix(dashboard): fix mobile layout on iphone', body: '…' }

migrateLegacyPrTitle('Task task-1: [Mission: X] Add OAuth callback');
// → 'feat(auth): add oauth callback'
```

## Type & scope inference

| Signal in goal/title | Inferred type |
|---------------------|---------------|
| fix, bug, hotfix, crash, regression | `fix` |
| refactor, restructure, rename, cleanup | `refactor` |
| document, readme, guide | `docs` |
| test, coverage, vitest | `test` |
| chore, deps, ci, lint | `chore` |
| *(default)* | `feat` |

| Signal | Inferred scope |
|--------|----------------|
| dashboard, mobile, ui | `dashboard` |
| github, git, branch, pr | `git` |
| loop-engine, critique, verify | `loop` |
| pm, team, blackboard | `pm` |
| mcp, cursor, agent | `mcp` |
| api, endpoint, route | `api` |
| auth, login, jwt | `auth` |
| *(default)* | `roland` |
