# Roland — Product Vision

Roland is a **production-grade agent looping harness** for software engineering. It combines a PM team orchestrator, structured iteration loops, evaluation gates, and clean GitHub workflows into a single platform operable from CLI, Cursor MCP, and a mobile-friendly command center.

---

## North star

> Reliable, iterative agent missions that **stop when done** — not when the token budget runs out.

Roland adopts patterns from [loops.elorm.xyz](https://loops.elorm.xyz):

- Self-paced iterations with explicit phase transitions
- Between-iteration sanity checks (`npm test`, `npm run lint`)
- Declarative **exit conditions** (gates green, confidence streak, command success)
- **Reflection memory** that compounds learnings across iterations
- Critique-driven retry with escalation to human operators

## Core pillars

### 1. Closed-Loop Harness

The primary execution model for production missions. Templates in `recipes/loops/` define phases, verification strategies, exit conditions, and between-iteration commands.

Entry point:

```bash
roland team "goal" --loop-template closed-loop-harness
```

See [guides/closed-loop-harness.md](guides/closed-loop-harness.md).

### 2. PM Team Engine

For larger missions, the Lead PM (`grok-4.3`) decomposes goals into parallel waves of specialist agents, reviews outputs, adjusts plans, and delivers a Mission Complete synthesis.

Execution-path triage routes small work to **Direct** (Cursor chat) and complex work to **Team** (`roland team`).

### 3. Clean PR Conventions

Every autogenerated PR uses conventional-commit titles and structured bodies. Legacy `Task task-N:` noise is stripped automatically. Batch migration via `roland pr-cleanup --apply`.

See [guides/pr-title-convention.md](guides/pr-title-convention.md).

### 4. Command Center (Dashboard 2.0)

`npm run serve-dashboard` serves a mobile-first command center:

- Live run state with WebSocket push
- Loop health and exit condition visibility
- HITL controls (pause, resume, inject, replan)
- **GitHub integration** — connect PAT, browse repos, one-click clone
- Project discovery and switching

Bind `--host 0.0.0.0` for Tailscale / LAN access from iPhone.

### 5. Cross-run memory

`.roland/memory.md` stores architecture decisions, coding standards, past mistakes, and preferences. Smart recall injects relevant bullets into PM planning prompts. Post-run retrospectives propose new memory bullets.

Loop-specific memory lives in `.roland/loops/<loop-id>/reflection.md`.

## Execution modes

| Mode | Best for | Entry |
|------|----------|-------|
| **Closed loop** | Iterative features with gates | `roland team --loop-template` |
| **Team** | Multi-file PM missions | `roland team "goal"` |
| **Direct** | Quick chat fixes | Cursor MCP tools |
| **Orchestrate** | SDK supervisor | `roland orchestrate "goal"` |
| **Background** | Long missions | `roland team --background` |

## State layout

```
.roland/
├── command-blackboard.md      # Human battlespace
├── blackboard.json            # Typed tasks / blockers
├── memory.md                  # Cross-run knowledge
├── run-state.json             # Live progress + loop phase
├── loop-state.json            # Active loop engine state
├── usage-history.json         # Token / cost tracking
└── loops/
    └── <loop-id>/
        ├── state.json         # Confidence streak, exit status
        ├── reflection.md      # Iteration learnings
        ├── checkpoints/       # Resume snapshots
        └── closed-loop-pr.json  # Formatted PR draft
```
